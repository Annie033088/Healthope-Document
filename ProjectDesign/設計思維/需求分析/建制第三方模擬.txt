建制第三方模擬

1.otp驗證模擬
2.QR Code 掃碼驗證
3.QR Code 掃碼報到
4.刷卡機回傳模擬
5.印電子發票模擬

－－－－－otp 驗證－－－－－
實現方式：
1.會員註冊資訊傳至後端
2.紀錄資訊(phone)產生對應OTP碼
3.發送至模擬第三方簡訊發送平台
4.特定頁面模擬手機收到簡訊通知(以輪巡簡單模擬效果)
5.輸入OTP碼驗證成功

機制: 
1.OTP碼 有效期為五分鐘, 時間一到廢棄此 OTP碼
2.OTP碼 五分鐘內不可重複發送 (前端使用 cookie 計時, 後端使用 redis 設定過期時間)

－－－－－QR Code 掃碼驗證－－－－－
實現方式：
1.會員出示 QR Code
2.對準 『自動感應式固定掃碼器』 掃碼
3.掃碼器將資料傳輸至電腦
QR Code 資訊設計：入場身份分類(1) + Id(10) + 前二者簽章後字串(64)
入場身份分類包括 1.會籍 2.教練 3.一次性票劵
4.後端透過加密法驗證是否正確
5.入場成功/失敗
(系統紀錄會員.教練入場資訊，或修改一次性票劵的狀態)

QR Code 模擬方式：
第一種 - 模擬頁面顯示 QR Code，並手動發送後，後端會接收到資料
第二種 - 模擬頁面掃描 QR Code，讀取後發送到後端，顯示發送成功 ✅

機制：
1.會籍.教練入場為動態刷新 QR Code, 生成的 QR Code 每 30 秒即無效 (QR Code 生成時存放在 redis)
2.一次性票劵為 紙張 QR Code, 僅購買當日可用 且 僅可使用一次 (購買時，當即列印 QR Code，並將資訊存在資料庫)

－－－－－QR Code 掃碼報到－－－－－
實現方式：
1.會員出示 QR Code
2.對準 『自動感應式固定掃碼器』 掃碼
3.掃碼器將資料傳輸至電腦
QR Code 資訊設計：Id(10) + 前者簽章後字串(64)
4.後端驗證
5.報到成功/失敗
(系統修改狀態)

機制：
1.生成的 QR Code 為一次性有效(存在 redis)

－－－－－刷卡機回傳模擬－－－－－
實現方式：
1.後台生成訂單
2.管理者選擇 現金/刷卡 付款
3.系統請求刷卡並等待結果 (傳送金額跟購買項目至模擬刷卡機)
4.模擬頁面收到商品及金額，在頁面輸入卡號，並點選確認
5.顯示結果(成功/失敗)，並回傳到後端
6.後端接收到授權碼、卡號、金額、結果、交易時間
7.若成功的話：新增一筆信用卡交易紀錄(資料包括 訂單外鍵、授權碼、卡片末四碼)，修改訂單狀態

交易紀錄用途：提供給管理者快速核對會員爭議的交易項目

－－－－－印電子發票模擬－－－－－
實現方式：
1.訂單付款成功後
2.資料庫生成電子發票資料
3.產生電子發票在模擬頁面上
4.可以用手機掃看看 QR Code , 印出資訊















補充知識：
－－－－－Virtual Serial Port Driver（虛擬串口驅動）－－－－－
✅ 它是什麼？
Virtual Serial Port Driver（簡稱 VSPD）是一套可以在 Windows 上建立「虛擬 COM 埠」的軟體工具。

✅ 能做什麼？
模擬兩個 COM 埠（例如 COM5 和 COM6）之間有一條資料線連著。

讓你在沒有實體串口裝置的情況下，模擬串口通訊。

測試設備、模擬資料傳輸、或在開發過程中進行串口通訊模擬。

✅ 怎麼運作？
建立一對虛擬串口：COM5 ↔ COM6

發送端程式連接 COM5，接收端程式連接 COM6

COM5 發送的資料會馬上「轉送」到 COM6，就像真的插著一條線。

✅ 系統能偵測到嗎？
✔️ 是的！虛擬埠會被 Windows 當作真的 COM 埠看待，你的程式能用一般方式掃描、連接、傳送接收資料。

－－－－－SSCOM 串口助手（Serial Port Tool）基礎知識－－－－－
✅ 它是什麼？
SSCOM 是一套常用的 串口調試工具，主要功能是用來測試與模擬串口資料傳輸的行為。

✅ 能做什麼？
開啟任一個 COM 埠（包括虛擬埠）

發送字串資料（手動或自動）

接收來自 COM 埠的資料並顯示出來

可以設定自動換行、自動重複發送、HEX 模式、延遲發送等。

✅ 常見用途？
模擬設備發送資料（例如模擬 QR code 掃描器）

測試你的程式是否能正確接收 COM 埠資料

調試嵌入式設備與電腦的串口連線

 如何搭配使用？

工具		角色		功能
VSPD	線		模擬兩個 COM 埠間的連線
SSCOM	掃描器	模擬掃描器發送資料（透過 COM5）
程式		接收器	接收模擬掃描器發送來的資料（透過 COM6）
